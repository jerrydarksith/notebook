# Notebook AI Rules

Ти працюєш над довготривалим проєктом.

Перед будь-якими змінами дотримуйся цих правил.

## Загальні правила

- Не змінюй архітектуру без дозволу.
- Не додавай нових бібліотек без пояснення та підтвердження.
- Не роби великих рефакторингів.
- Не видаляй файли без необхідності.
- Не змінюй код, який не стосується поточного завдання.

## Архітектура

- Python 3.14
- SQLite
- Repository Pattern
- Service Layer
- Telegram є лише інтерфейсом.
- Бізнес-логіка тільки в Service.
- SQL тільки в Repository.
- Telegram handlers не повинні містити бізнес-логіку.

## Запуск

Для запуску використовуй:

PYTHONPATH=src .venv/bin/python -m personal_bot

Для тестів:

.venv/bin/python -m unittest discover tests

Не використовуй команду:

python

оскільки вона відсутня.

## Перед початком роботи

Спочатку проаналізуй лише ті файли, які потрібні для поточного завдання.

Не аналізуй увесь проєкт без необхідності.

## Після завершення

Обов'язково:

- покажи список створених файлів;
- покажи список змінених файлів;
- виконай тести;
- виконай code review;
- зупинись і дочекайся підтвердження.
